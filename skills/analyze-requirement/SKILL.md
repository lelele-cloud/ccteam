# 需求分析技能

系统化分析用户需求，理解业务目标，识别核心功能点，输出结构化的需求分析报告。

## 适用角色

- 产品经理（主要）
- 项目经理
- 技术负责人

## 输入/输出

**输入:** 需求来源、背景信息、干系人信息、参考文档

**输出:**
- 需求分析报告：`docs/ccteam/01-requirements/requirement-analysis.md`
- 需求清单
- 澄清问题列表

## 执行步骤

### 步骤1：需求理解

1. 收集原始需求，记录关键术语
2. 识别干系人及关注点
3. 理解业务背景和目标

### 步骤2：需求澄清

1. 提出澄清问题
2. 验证假设条件
3. 确认边界条件

### 步骤3：需求拆解

1. 功能分解为独立模块
2. 用例分析
3. 提取非功能需求（性能、安全、可用性）

### 步骤4：优先级评估

使用MoSCoW方法：Must/Should/Could/Won't

评估维度：业务价值(40%)、用户影响(25%)、技术复杂度(20%)、依赖关系(15%)

### 步骤5：输出报告

详细报告模板见 [references/report-template.md](references/report-template.md)

## 质量检查

- [ ] 所有原始需求已分析记录
- [ ] 功能和非功能需求都已覆盖
- [ ] 需求描述无歧义
- [ ] 优先级评估合理
