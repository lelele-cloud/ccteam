# CCTeam - 新功能开发

在现有项目上添加新功能，流程可根据功能复杂度进行裁剪。

## 使用方法

### 1. 收集功能信息

```
请提供功能信息：

1. 功能名称：
2. 功能描述：
3. 涉及模块：
4. 是否需要UI设计：是/否
5. 是否需要数据库变更：是/否
6. 是否需要新API：是/否
```

### 2. 流程裁剪

根据功能需求自动裁剪流程：

| 功能类型 | 包含阶段 |
|----------|----------|
| 纯后端功能 | 需求→技术评审→数据库→API→后端→测试 |
| 纯前端功能 | 需求→UI/UX→前端→测试 |
| 全栈功能 | 完整10阶段 |
| Bug修复 | 分析→修复→测试 |

### 3. 初始化

1. 读取现有 `project-status.json`
2. 创建功能分支记录
3. 启动第一个适用阶段

### 4. 执行开发

按裁剪后的流程执行，跳过不需要的阶段。

## 与新项目的区别

| 方面 | 新项目 | 新功能 |
|------|--------|--------|
| 流程 | 完整10阶段 | 可裁剪 |
| 文档 | 创建新目录 | 追加到现有目录 |
| 技术栈 | 需要选择 | 沿用现有 |
| 评审 | 严格 | 可简化 |
