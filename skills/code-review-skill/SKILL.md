# 代码审查技能

系统化地进行代码审查，确保代码质量和团队规范一致性。

## 适用角色

- 代码审查员（主要）
- 技术负责人
- 高级工程师

## 输入/输出

**输入:** Pull Request、代码变更、相关文档

**输出:** 审查意见、改进建议

## 审查维度

### 1. 功能正确性

- [ ] 实现符合需求
- [ ] 边界条件处理
- [ ] 异常处理完整

### 2. 代码质量

- [ ] 命名清晰有意义
- [ ] 函数职责单一
- [ ] 无重复代码
- [ ] 复杂度可控

### 3. 安全性

- [ ] 无SQL注入风险
- [ ] 无XSS风险
- [ ] 敏感信息处理正确
- [ ] 权限检查完整

### 4. 性能

- [ ] 无N+1查询
- [ ] 无内存泄漏
- [ ] 无不必要的计算

### 5. 可维护性

- [ ] 有必要的注释
- [ ] 测试覆盖充分
- [ ] 文档更新

## 反馈原则

- 对事不对人
- 给出具体建议
- 解释原因
- 区分必须修改和建议
