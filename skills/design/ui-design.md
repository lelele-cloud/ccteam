# UI设计技能

## 概述

系统化进行用户界面设计，从设计系统建立到界面输出，确保设计的一致性、美观性和可用性。

## 适用角色

- UI设计师（主要）
- UX设计师
- 前端工程师（参考）

## 输入

- **产品需求**：PRD、用户故事
- **交互设计**：交互稿、用户流程图
- **品牌资料**：Logo、品牌色、视觉规范
- **现有设计**：已有设计系统（如有）

## 输出

- **设计系统**：`docs/ccteam/03-design/design-system.md`
- **界面设计**：`docs/ccteam/03-design/ui-specs/`
- **组件库**：`docs/ccteam/03-design/components.md`
- **设计标注**：切图资源和标注说明

## 执行步骤

### 步骤1：设计准备

1. **品牌分析**
   - 收集品牌资料（Logo、颜色、字体）
   - 理解品牌调性和定位
   - 明确视觉风格方向

2. **竞品研究**
   - 分析3-5个竞品界面
   - 记录设计亮点和不足
   - 提炼设计灵感

3. **约束确认**
   - 目标平台（Web/Mobile）
   - 适配要求（屏幕尺寸）
   - 技术限制（前端框架）

### 步骤2：设计系统建立

1. **颜色系统**
   ```markdown
   ## 颜色定义

   ### 品牌色
   - 主色: #1890FF (用于主要操作、链接)
   - 辅助色: #52C41A (用于成功状态)

   ### 功能色
   - 成功: #52C41A
   - 警告: #FAAD14
   - 错误: #FF4D4F
   - 信息: #1890FF

   ### 中性色
   - 标题: #262626
   - 正文: #595959
   - 次要: #8C8C8C
   - 禁用: #BFBFBF
   - 边框: #D9D9D9
   - 分割: #F0F0F0
   - 背景: #F5F5F5
   ```

2. **字体系统**
   ```markdown
   ## 字体定义

   ### 字体族
   - 中文: PingFang SC, Microsoft YaHei
   - 英文: -apple-system, Helvetica Neue
   - 代码: SFMono-Regular, Consolas

   ### 字号层级
   | 用途 | 字号 | 行高 | 字重 |
   |------|------|------|------|
   | H1标题 | 30px | 38px | 600 |
   | H2标题 | 24px | 32px | 600 |
   | H3标题 | 20px | 28px | 600 |
   | H4标题 | 16px | 24px | 600 |
   | 正文 | 14px | 22px | 400 |
   | 辅助文字 | 12px | 20px | 400 |
   ```

3. **间距系统**
   ```markdown
   ## 间距定义

   基础单位: 4px

   | 名称 | 值 | 用途 |
   |------|-----|------|
   | xs | 4px | 图标与文字间距 |
   | sm | 8px | 紧凑元素间距 |
   | md | 16px | 标准元素间距 |
   | lg | 24px | 分组间距 |
   | xl | 32px | 区块间距 |
   | xxl | 48px | 大区块间距 |
   ```

### 步骤3：组件设计

1. **基础组件**
   ```markdown
   ## 按钮组件

   ### 类型
   - 主按钮: 用于主要操作
   - 次按钮: 用于次要操作
   - 文字按钮: 用于轻量操作
   - 图标按钮: 用于空间受限场景

   ### 尺寸
   - 大: 40px高度
   - 中: 32px高度（默认）
   - 小: 24px高度

   ### 状态
   - 默认 / 悬停 / 按下 / 禁用 / 加载中
   ```

2. **复合组件**
   - 表单组件（输入框、选择器、日期选择）
   - 导航组件（菜单、面包屑、标签页）
   - 数据展示（表格、卡片、列表）
   - 反馈组件（消息提示、对话框、通知）

### 步骤4：界面设计

1. **页面布局**
   ```markdown
   ## 布局规范

   ### 栅格系统
   - 总宽度: 1200px
   - 列数: 24列
   - 列间距: 16px

   ### 常用布局
   - 上下布局: 顶部导航 + 内容区
   - 左右布局: 侧边导航 + 内容区
   - 混合布局: 顶部导航 + 侧边导航 + 内容区
   ```

2. **设计规范**
   - 对齐：元素左对齐为主
   - 留白：足够的呼吸空间
   - 层次：通过大小、颜色、粗细区分
   - 一致：相同元素保持一致

### 步骤5：设计交付

1. **设计标注**
   - 尺寸标注（宽高、间距）
   - 颜色标注（色值、透明度）
   - 字体标注（字号、字重、行高）
   - 状态说明（各状态效果）

2. **切图规范**
   - 格式：PNG（带透明）/ SVG（图标）
   - 尺寸：1x、2x、3x
   - 命名：模块_类型_名称_状态

## 质量检查

### 一致性检查
- [ ] 颜色使用符合规范
- [ ] 字体使用符合规范
- [ ] 间距使用符合规范
- [ ] 组件使用正确

### 美观性检查
- [ ] 整体视觉协调
- [ ] 配色和谐
- [ ] 排版整齐
- [ ] 细节精致

### 可用性检查
- [ ] 层次分明
- [ ] 重点突出
- [ ] 操作引导清晰
- [ ] 状态反馈明确

### 交付检查
- [ ] 标注完整准确
- [ ] 切图规范齐全
- [ ] 设计说明清晰

## 常见问题与解决方案

| 问题 | 解决方案 |
|------|----------|
| 颜色不协调 | 使用60-30-10配色法则 |
| 层次不清 | 增加对比度（大小/颜色/粗细） |
| 视觉杂乱 | 减少元素，增加留白 |
| 风格不统一 | 建立并遵循设计系统 |
