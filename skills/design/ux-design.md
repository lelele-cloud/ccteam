# UX设计技能

## 概述

系统化进行用户体验设计，从用户研究到交互设计，确保产品易用、高效、令人愉悦。

## 适用角色

- UX设计师（主要）
- 产品经理
- UI设计师（参考）

## 输入

- **产品需求**：PRD、用户故事
- **用户数据**：用户画像、使用数据
- **业务目标**：转化目标、关键指标
- **竞品分析**：竞品体验分析

## 输出

- **用户流程图**：`docs/ccteam/03-design/user-flows/`
- **交互原型**：`docs/ccteam/03-design/prototypes/`
- **交互规范**：`docs/ccteam/03-design/interaction-spec.md`
- **可用性报告**：`docs/ccteam/03-design/usability-report.md`

## 执行步骤

### 步骤1：用户研究

1. **用户画像**
   ```markdown
   ## 用户画像: 小白用户

   ### 基本信息
   - 年龄: 25-35岁
   - 职业: 办公室白领
   - 技术水平: 中等
   - 使用频率: 每天

   ### 目标和动机
   - 主要目标: 快速完成任务
   - 次要目标: 了解进度
   - 动机: 提高工作效率

   ### 痛点和需求
   - 痛点: 操作复杂、难以上手
   - 需求: 简单直观、有引导
   - 期望: 5分钟内完成核心任务
   ```

2. **用户旅程**
   ```
   阶段: 发现 → 注册 → 首次使用 → 持续使用 → 推荐
         │       │         │           │         │
   行为: 搜索   填写表单   功能探索    日常使用   分享
         │       │         │           │         │
   情绪: 好奇   期待      困惑→理解   满意       认可
         │       │         │           │         │
   机会: 精准定位 简化流程  新手引导   持续优化   激励机制
   ```

### 步骤2：信息架构

1. **内容组织**
   ```
   一级导航:
   ├── 首页 (Dashboard)
   ├── 项目 (Projects)
   │   ├── 项目列表
   │   ├── 项目详情
   │   └── 项目设置
   ├── 团队 (Team)
   │   ├── 成员管理
   │   └── 权限设置
   └── 设置 (Settings)
       ├── 个人设置
       └── 系统设置
   ```

2. **导航设计原则**
   - 层级不超过3层
   - 命名清晰易懂
   - 位置符合用户预期
   - 支持快捷导航

### 步骤3：交互设计

1. **用户流程**
   ```markdown
   ## 创建项目流程

   步骤1: 点击"创建项目"按钮
          ↓
   步骤2: 填写项目基本信息
          - 项目名称（必填）
          - 项目描述（选填）
          - 项目类型（必选）
          ↓
   步骤3: 选择项目成员（可跳过）
          ↓
   步骤4: 确认创建
          ↓
   步骤5: 进入项目详情页
   ```

2. **交互规范**
   ```markdown
   ## 交互规范

   ### 反馈原则
   - 操作后100ms内有响应
   - 加载超过1s显示加载状态
   - 成功/失败有明确提示
   - 危险操作需要确认

   ### 表单交互
   - 实时验证+提交验证
   - 错误提示就近显示
   - 必填项标识清晰
   - 支持键盘导航

   ### 导航交互
   - 当前位置有明确标识
   - 返回操作一键可达
   - 层级面包屑导航
   - 支持快捷键
   ```

### 步骤4：原型制作

1. **原型层次**
   | 层次 | 目的 | 保真度 | 工具 |
   |------|------|--------|------|
   | 草图 | 快速探索 | 低 | 纸笔 |
   | 线框 | 验证流程 | 中 | Figma/Axure |
   | 原型 | 用户测试 | 高 | Figma |

2. **原型规范**
   - 覆盖核心流程
   - 包含主要状态
   - 标注交互说明
   - 可点击演示

### 步骤5：可用性测试

1. **测试计划**
   ```markdown
   ## 可用性测试计划

   ### 测试目标
   - 验证核心流程可用性
   - 发现交互问题
   - 收集改进建议

   ### 测试任务
   1. 完成注册流程（预计3分钟）
   2. 创建一个新项目（预计2分钟）
   3. 邀请团队成员（预计1分钟）

   ### 测试指标
   - 任务完成率
   - 任务完成时间
   - 错误次数
   - 用户满意度（SUS评分）
   ```

2. **测试执行**
   - 招募5-8名用户
   - 观察用户操作
   - 记录问题和反馈
   - 分析结果并优化

## 质量检查

### 用户研究检查
- [ ] 用户画像清晰
- [ ] 用户需求明确
- [ ] 痛点已识别
- [ ] 机会点已记录

### 信息架构检查
- [ ] 结构层级合理
- [ ] 命名易于理解
- [ ] 导航符合预期
- [ ] 内容组织清晰

### 交互设计检查
- [ ] 流程清晰高效
- [ ] 操作有反馈
- [ ] 错误可恢复
- [ ] 状态可感知

### 原型检查
- [ ] 覆盖核心流程
- [ ] 状态完整
- [ ] 可演示交互
- [ ] 有说明文档

### 可用性检查
- [ ] 任务可完成
- [ ] 效率可接受
- [ ] 错误可预防
- [ ] 满意度达标

## 常见问题与解决方案

| 问题 | 解决方案 |
|------|----------|
| 用户难以上手 | 添加新手引导、简化首次体验 |
| 操作步骤多 | 合并步骤、智能默认值 |
| 反馈不及时 | 添加即时反馈、进度指示 |
| 找不到功能 | 优化导航、添加搜索 |
