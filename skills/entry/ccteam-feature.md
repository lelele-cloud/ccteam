# CCTeam - 新功能开发

## 概述

在现有项目上添加新功能，流程更精简，可跳过不需要的阶段。

## 使用方法

当用户调用 `/ccteam-feature` 时：

### 1. 收集功能信息

```
请提供功能信息：

1. 功能名称：
2. 功能描述：
3. 涉及的模块：
4. 是否需要新的UI设计？ (是/否)
5. 是否需要数据库变更？ (是/否)
6. 是否需要新的API？ (是/否)
```

### 2. 确定流程阶段

根据功能需求确定需要的阶段：

| 条件 | 需要的阶段 |
|------|-----------|
| 基础 | 需求分析 → 技术评审 → 开发 → 测试 |
| +UI设计 | + UI/UX设计阶段 |
| +数据库 | + 数据库设计阶段 |
| +API | + API设计阶段 |

### 3. 简化流程示例

**纯前端功能**：
```
需求分析 → 技术评审 → 前端开发 → 测试 → 部署
```

**带API的功能**：
```
需求分析 → 技术评审 → API设计 → 前后端开发 → 测试 → 部署
```

**完整功能**：
```
需求分析 → 技术评审 → UI/UX → 数据库 → API → 开发 → 测试 → 部署
```

### 4. 启动流程

1. 确认跳过的阶段
2. 更新 `project-status.json`
3. 从第一个需要的阶段开始

## 与新项目的区别

- 可跳过不需要的阶段
- 文档要求更简洁
- 关注与现有系统的兼容性
- 更注重回归测试
