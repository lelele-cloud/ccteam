# 评审门禁 (Review Gate)

每个阶段完成时的评审检查点，确保产出物质量符合标准。

## 评审模式

| 模式 | 说明 |
|------|------|
| single | 单人评审通过即可 |
| majority | 2/3多数通过 |
| unanimous | 全票通过 |

## 评审流程

1. **产出物检查**：验证必需的产出物是否存在
2. **质量检查**：按检查清单验证质量
3. **评审决策**：通过/不通过
4. **结果处理**：更新状态、记录反馈

## 各阶段评审标准

| 阶段 | 产出物 | 评审人 | 重点检查 |
|------|--------|--------|----------|
| 需求分析 | PRD | 技术负责人 | 完整性、可行性 |
| 技术评审 | 架构文档 | 架构师 | 合理性、扩展性 |
| UI/UX设计 | 设计稿 | 产品经理 | 一致性、体验 |
| 数据库设计 | ER图、DDL | 后端负责人 | 规范性、性能 |
| API设计 | OpenAPI | 前端负责人 | 完整性、易用性 |
| 开发 | 代码 | 代码审查员 | 质量、规范 |
| 测试 | 测试报告 | 技术负责人 | 覆盖率、通过率 |
| 部署 | 部署文档 | 运维负责人 | 可操作性 |

## 评审结果处理

**通过**：
- 记录评审意见
- 标记阶段完成
- 解锁下一阶段

**不通过**：
- 创建Issue记录问题
- 标记具体修改建议
- 回退给负责人修改
