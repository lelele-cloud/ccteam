# 测试执行技能

按照测试计划系统化地执行测试，记录结果并输出测试报告。

## 适用角色

- 测试工程师（主要）
- 开发工程师（单元测试）

## 输入/输出

**输入:** 测试计划、测试用例、待测系统

**输出:**
- 测试报告：`docs/ccteam/08-testing/test-report.md`
- 缺陷清单

## 执行流程

### 1. 准备阶段

- 确认测试环境就绪
- 准备测试数据
- 检查测试工具

### 2. 执行阶段

- 按优先级执行用例
- 记录测试结果
- 发现缺陷立即记录

### 3. 缺陷管理

| 级别 | 定义 | 修复时限 |
|------|------|----------|
| P0-致命 | 系统崩溃 | 4小时 |
| P1-严重 | 主功能异常 | 24小时 |
| P2-一般 | 次功能异常 | 3天 |
| P3-轻微 | 体验问题 | 下个版本 |

### 4. 报告阶段

- 汇总测试结果
- 分析缺陷分布
- 评估测试覆盖率
- 给出质量评估

## 测试报告内容

1. 执行概要
2. 测试范围
3. 测试结果统计
4. 缺陷分析
5. 覆盖率分析
6. 风险评估
7. 质量建议
