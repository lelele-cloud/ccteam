# 测试计划技能

制定全面的测试策略和计划，确保软件质量目标的达成。

## 适用角色

- 测试工程师（主要）
- 测试负责人
- 项目经理

## 输入/输出

**输入:** PRD、技术设计文档、用户故事、项目计划

**输出:** 测试计划文档：`docs/ccteam/08-testing/test-plan.md`

## 测试金字塔

```
        / E2E \         10% - 端到端测试
       /______\
      /Integration\     20% - 集成测试
     /______________\
    /   Unit Tests   \  70% - 单元测试
```

## 测试类型

| 类型 | 目的 | 执行频率 |
|------|------|----------|
| 单元测试 | 验证最小单元 | 每次提交 |
| 集成测试 | 验证模块协作 | 每次构建 |
| E2E测试 | 验证完整流程 | 每日/部署前 |
| 性能测试 | 验证性能指标 | 发布前 |
| 安全测试 | 验证安全性 | 发布前 |

## 计划内容

1. 测试策略
2. 测试范围
3. 测试环境
4. 资源需求
5. 进度安排
6. 风险评估
7. 准入准出标准
