# UI设计技能

系统化进行用户界面设计，从设计系统建立到界面输出，确保设计的一致性、美观性和可用性。

## 适用角色

- UI设计师（主要）
- UX设计师
- 前端工程师（参考）

## 输入/输出

**输入:** PRD、交互设计、品牌资料、现有设计系统

**输出:**
- 设计系统: `docs/ccteam/03-design/design-system.md`
- 界面设计: `docs/ccteam/03-design/ui-specs/`
- 组件库: `docs/ccteam/03-design/components.md`

## 工作流程

```
设计准备 → 设计系统建立 → 组件设计 → 界面设计 → 设计交付
```

### 步骤1: 设计准备

1. 收集品牌资料（Logo、颜色、字体）
2. 分析3-5个竞品界面
3. 确认目标平台和适配要求

### 步骤2: 设计系统建立

**颜色系统:**

| 类型 | 颜色 | 用途 |
|------|------|------|
| 主色 | #1890FF | 主要操作、链接 |
| 成功 | #52C41A | 成功状态 |
| 警告 | #FAAD14 | 警告状态 |
| 错误 | #FF4D4F | 错误状态 |
| 标题 | #262626 | 标题文字 |
| 正文 | #595959 | 正文文字 |

**字体系统:**

| 用途 | 字号 | 行高 | 字重 |
|------|------|------|------|
| H1标题 | 30px | 38px | 600 |
| H2标题 | 24px | 32px | 600 |
| 正文 | 14px | 22px | 400 |
| 辅助文字 | 12px | 20px | 400 |

详细规范见 [references/design-tokens.md](references/design-tokens.md)

### 步骤3: 组件设计

基础组件（按钮、输入框、选择器）和复合组件（表单、导航、数据展示）。

组件规范见 [references/components.md](references/components.md)

### 步骤4: 界面设计

- 栅格系统: 1200px总宽，24列
- 对齐: 元素左对齐为主
- 留白: 足够的呼吸空间
- 层次: 通过大小、颜色区分

### 步骤5: 设计交付

- 尺寸标注（宽高、间距）
- 颜色标注（色值、透明度）
- 切图规范（PNG/SVG，1x/2x/3x）

## 质量检查

**一致性:**
- [ ] 颜色使用符合规范
- [ ] 字体使用符合规范
- [ ] 间距使用符合规范

**美观性:**
- [ ] 整体视觉协调
- [ ] 配色和谐
- [ ] 排版整齐

**交付:**
- [ ] 标注完整准确
- [ ] 切图规范齐全

## 常见陷阱

| 陷阱 | 规避方法 |
|------|----------|
| 颜色不协调 | 使用60-30-10配色法则 |
| 层次不清 | 增加对比度 |
| 视觉杂乱 | 减少元素，增加留白 |
| 风格不统一 | 建立并遵循设计系统 |
