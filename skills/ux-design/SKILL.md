# UX设计技能

系统化进行用户体验设计，从用户研究到交互设计，确保产品易用、高效、令人愉悦。

## 适用角色

- UX设计师（主要）
- 产品经理
- UI设计师（参考）

## 输入/输出

**输入:** PRD、用户画像、业务目标、竞品分析

**输出:**
- 用户流程图: `docs/ccteam/03-design/user-flows/`
- 交互原型: `docs/ccteam/03-design/prototypes/`
- 交互规范: `docs/ccteam/03-design/interaction-spec.md`
- 可用性报告: `docs/ccteam/03-design/usability-report.md`

## 工作流程

```
用户研究 → 信息架构 → 交互设计 → 原型制作 → 可用性测试
```

### 步骤1: 用户研究

**用户画像模板:**
```markdown
## 用户画像: 小白用户

### 基本信息
- 年龄: 25-35岁
- 职业: 办公室白领
- 技术水平: 中等

### 目标和动机
- 主要目标: 快速完成任务
- 动机: 提高工作效率

### 痛点和需求
- 痛点: 操作复杂、难以上手
- 需求: 简单直观、有引导
```

**用户旅程:**
```
发现 → 注册 → 首次使用 → 持续使用 → 推荐
```

详细模板见 [references/user-research.md](references/user-research.md)

### 步骤2: 信息架构

- 层级不超过3层
- 命名清晰易懂
- 位置符合用户预期

### 步骤3: 交互设计

**交互规范:**
- 操作后100ms内有响应
- 加载超过1s显示加载状态
- 成功/失败有明确提示
- 危险操作需要确认

详细规范见 [references/interaction-spec.md](references/interaction-spec.md)

### 步骤4: 原型制作

| 层次 | 目的 | 保真度 |
|------|------|--------|
| 草图 | 快速探索 | 低 |
| 线框 | 验证流程 | 中 |
| 原型 | 用户测试 | 高 |

### 步骤5: 可用性测试

测试计划见 [references/usability-test.md](references/usability-test.md)

## 质量检查

**用户研究:**
- [ ] 用户画像清晰
- [ ] 用户需求明确

**信息架构:**
- [ ] 结构层级合理
- [ ] 导航符合预期

**交互设计:**
- [ ] 流程清晰高效
- [ ] 操作有反馈

## 常见陷阱

| 陷阱 | 规避方法 |
|------|----------|
| 用户难以上手 | 添加新手引导 |
| 操作步骤多 | 合并步骤、智能默认值 |
| 反馈不及时 | 添加即时反馈 |
| 找不到功能 | 优化导航、添加搜索 |
