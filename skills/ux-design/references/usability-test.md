# 可用性测试指南

## 测试计划模板

```markdown
## 可用性测试计划

### 测试目标
- 验证核心流程可用性
- 发现交互问题
- 收集改进建议

### 测试任务
1. 完成注册流程（预计3分钟）
2. 创建一个新项目（预计2分钟）
3. 邀请团队成员（预计1分钟）

### 测试指标
- 任务完成率
- 任务完成时间
- 错误次数
- 用户满意度（SUS评分）

### 测试人数
- 5-8名用户（发现80%问题）
```

## 测试执行

1. **准备阶段**
   - 准备测试环境
   - 编写测试脚本
   - 准备录屏工具

2. **执行阶段**
   - 欢迎和说明
   - 执行测试任务
   - 观察用户操作
   - 记录问题和反馈

3. **分析阶段**
   - 整理测试数据
   - 分析问题原因
   - 提出改进建议

## SUS评分量表

System Usability Scale (系统可用性量表)

1. 我想经常使用这个系统
2. 我觉得这个系统不必要地复杂
3. 我认为这个系统易于使用
4. 我认为我需要技术支持才能使用这个系统
5. 我发现系统的各种功能整合得很好
6. 我认为这个系统有太多的不一致性
7. 我想大多数人会很快学会使用这个系统
8. 我发现这个系统使用起来非常繁琐
9. 我对使用这个系统感到很有信心
10. 在使用这个系统之前我需要学习很多东西

评分: 1(强烈不同意) - 5(强烈同意)
SUS分数 > 68 为可接受
