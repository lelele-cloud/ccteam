# PRD编写技能

根据需求分析结果，编写标准化的产品需求文档(PRD)。

## 适用角色

- 产品经理（主要）

## 输入/输出

**输入:** 需求分析报告、业务目标、用户反馈

**输出:** PRD文档：`docs/ccteam/01-requirements/prd.md`

## PRD结构

1. **文档信息**：版本、状态、负责人
2. **产品概述**：背景、目标、范围
3. **用户分析**：用户画像、用户场景
4. **功能需求**：功能列表、详细描述、用例
5. **非功能需求**：性能、安全、兼容性
6. **交互设计**：页面流程、交互说明
7. **验收标准**：功能验收、质量标准
8. **附录**：术语、参考文档

## 写作原则

- 清晰：避免歧义
- 完整：覆盖所有场景
- 可测试：验收标准明确
- 可追溯：需求有编号

## 质量检查

- [ ] 功能描述清晰完整
- [ ] 用例覆盖主要场景
- [ ] 验收标准可测试
- [ ] 与技术团队对齐
