# 用户故事编写技能

将需求转化为用户故事格式，便于敏捷开发团队理解和执行。

## 适用角色

- 产品经理（主要）
- 技术负责人

## 输入/输出

**输入:** PRD、需求分析报告

**输出:** 用户故事：`docs/ccteam/01-requirements/user-stories.md`

## 用户故事格式

```
作为 [用户角色]
我想要 [功能/目标]
以便于 [价值/原因]

验收标准：
- Given [前置条件]
- When [操作]
- Then [预期结果]
```

## INVEST原则

- **I**ndependent：独立
- **N**egotiable：可协商
- **V**aluable：有价值
- **E**stimable：可估算
- **S**mall：小粒度
- **T**estable：可测试

## 故事拆分技巧

1. 按工作流步骤拆分
2. 按业务规则拆分
3. 按数据变体拆分
4. 按接口拆分
5. 按操作类型拆分（CRUD）

## 质量检查

- [ ] 符合INVEST原则
- [ ] 验收标准明确
- [ ] 故事粒度适中
- [ ] 依赖关系清晰
