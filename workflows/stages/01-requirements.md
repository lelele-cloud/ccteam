# 阶段1：需求分析

## 基本信息

| 属性 | 值 |
|------|-----|
| 阶段ID | 1 |
| 阶段名称 | requirements |
| 显示名称 | 需求分析 |
| 负责角色 | product-manager |
| 评审方 | tech-lead |
| 可跳过 | 否 |

## 阶段目标

- 明确项目背景和目标
- 定义目标用户群体
- 梳理核心功能需求
- 制定功能优先级
- 定义验收标准

## 输入要求

- 项目基本信息（名称、描述）
- 用户提供的需求描述
- 参考资料（竞品、设计稿等）

## 输出产物

| 产物 | 模板 | 存放位置 |
|------|------|----------|
| PRD文档 | `templates/prd.md` | `docs/ccteam/01-requirements/prd.md` |
| 用户故事 | `templates/user-story.md` | `docs/ccteam/01-requirements/user-stories.md` |
| 功能清单 | `templates/feature-list.md` | `docs/ccteam/01-requirements/feature-list.md` |

## 执行流程

```
1. 加载产品经理智能体
2. 分析用户需求描述
3. 与用户确认需求细节
4. 撰写PRD文档
5. 编写用户故事
6. 整理功能清单
7. 提交评审
```

## 评审检查清单

- [ ] 项目背景描述清晰
- [ ] 目标用户定义明确
- [ ] 核心功能列表完整
- [ ] 功能优先级已排序（P0/P1/P2/P3）
- [ ] 验收标准可量化
- [ ] 异常场景已考虑
- [ ] 非功能性需求已定义

## 评审通过条件

- 所有检查项通过
- 技术负责人确认可行性
- 无遗留问题

## 下一阶段

评审通过后，自动进入 **阶段2：技术评审**
